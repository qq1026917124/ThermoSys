# ThermoSys 架构文档

## 一、总体架构

Thermo-Agent Operating System 分为五层：

```
┌─────────────────────────────────────────┐
│           元认知层 (Meta-Cognition)        │
│  ┌──────────┐  ┌──────────┐            │
│  │ 反思Agent │  │ 进化Agent │            │
│  └──────────┘  └──────────┘            │
├─────────────────────────────────────────┤
│           决策层 (Decision)              │
│  ┌──────────┐  ┌──────────┐            │
│  │ 策略网络 │  │ 元控制器 │            │
│  └──────────┘  └──────────┘            │
├─────────────────────────────────────────┤
│           认知层 (Cognition)             │
│  ┌──────────────────────────┐          │
│  │      热力学世界模型       │          │
│  └──────────────────────────┘          │
├─────────────────────────────────────────┤
│           感知层 (Perception)            │
│  ┌──────────┐  ┌──────────┐            │
│  │ 数据抓取 │  │ 状态编码 │            │
│  └──────────┘  └──────────┘            │
├─────────────────────────────────────────┤
│           数据层 (Data)                  │
│  东方财富 / 雪球 / 百度指数 / 股吧       │
└─────────────────────────────────────────┘
```

## 二、热力学状态设计

### 2.1 状态向量

```python
ThermoState(
    ipv: float,              # 信息传播速度
    clarity: float,          # 路径清晰度
    coherence: float,        # 序参量
    sector_temperatures: Dict,  # 板块温度场
    group_phases: Dict,      # 群体相位
    entropy: float,          # 香农熵
    entropy_change: float,   # 熵减率
    rsi: float,              # 散户情绪
    mts: float,              # 热力态指数
)
```

### 2.2 状态编码器

`ThermoStateEncoder` 使用多模态融合架构：

- **文本编码器**：Transformer编码股吧/雪球文本
- **流向编码器**：LSTM编码资金流向序列
- **搜索编码器**：MLP编码搜索指数

输出统一的热力学状态张量，供Agent消费。

## 三、核心算法

### 3.1 散户情绪指数 (RSI)

```
RSI = 0.35 * Z(Δ融资余额) 
    + 0.25 * Z(小单净流入) 
    + 0.20 * Z(新增开户数) 
    + 0.15 * Z(搜索指数变化) 
    + 0.05 * Z(期权PCR)
```

**客观性保障**：
- Z-score标准化消除量纲差异
- 滚动分位数动态阈值适应市场结构
- 月度滚动回归动态校准权重

### 3.2 信息传播速度 (IPV)

```
v(t) = |∇ρ| / ρ * D / (1 + τ_lag)

ρ = 0.4 * Z(mentions) + 0.3 * Z(engagement) + 0.3 * Z(Δvolume)
```

**关键判定**：
- R₀ > 2：信息进入指数级扩散
- R₀ < 1：热点自生自灭

### 3.3 热传导网络

```
R_ij = 1 / (w_supply + w_fund + w_retail)

ΔT_i = α * Σ_j (1/R_ij) * (T_j - T_i) + S_i
```

**路径清晰度**：
- Clarity > 0.7：热点沿产业链有序传导，行情有纵深
- Clarity < 0.3：热点无序跳跃，缺乏主线

### 3.4 合力指标

**Kuramoto序参量**：
```
r(t) = |Σ w_j * e^(iθ_j)| / Σ w_j
```

**资金-信息共振度**：
```
Resonance = Corr(散户言论热度, 次日小单净流入, 20日)
```

**相变判定**：
- r > 0.6 + Resonance > 0.5：趋势性合力形成
- r < 0.3：市场混沌

## 四、Agent设计

### 4.1 世界模型

使用JEPA架构：
- 预测下一热力学状态
- 异方差不确定性估计
- 支持MC Dropout采样

### 4.2 策略网络

SAC风格Actor-Critic：
- Actor：输出高斯策略（仓位比例）
- 双Critic：分别评估财务收益和热力学稳定性
- 熵正则化鼓励探索

### 4.3 热力学约束奖励

```python
r = r_financial 
    - position_size * max(0, 0.3 - clarity) * 5.0   # 清晰度惩罚
    + 0.2 * (-entropy_change) * position              # 熵减奖励
    + 0.3 * coherence * position                      # 合力奖励
```

### 4.4 元学习控制器

MAML-based：
- LSTM识别市场Regime（牛市/熊市/震荡/过渡）
- 每个Regime对应一组快速适应参数
- 50交易日外循环，10日内循环

## 五、自我优化机制

### 5.1 反思Agent

**分析维度**：
1. 世界模型预测误差（分维度统计）
2. 策略亏损根因（亏损时的热力学状态共性）
3. 市场体制匹配度

**输出**：可执行的改进任务队列

### 5.2 进化Agent

**任务类型**：
- `feature_engineering`：添加新特征
- `architecture`：修改网络结构
- `hyperparameter`：超参搜索（Optuna）
- `reward_shaping`：调整奖励函数
- `constraint`：收紧约束条件

**验证流程**：
1. 隔离环境执行修改
2. 样本外回测（2020-2022年）
3. 夏普提升 > 0.1 才合并
4. 失败自动回滚

### 5.3 数据飞轮

```
每日数据 → 热力学状态 → Agent决策 → 交易执行
    ↑                                    |
    └──── 结果反馈 ← 世界模型更新 ← 策略训练

每周 → 反思Agent分析 → 生成改进任务
     → 进化Agent执行 → A/B测试 → 合并/回滚
```

## 六、回测框架

### 6.1 执行规则

- T日信号基于T日收盘数据
- T+1日开盘执行
- 支持滑点和佣金

### 6.2 仓位管理

默认策略：
- RSI < -2.0 + Coherence > 0.6：满仓
- RSI > 2.0 + Coherence < 0.3：空仓/做空
- 清晰度 < 0.3：仓位衰减
- 高熵状态：仓位限制

### 6.3 滚动前向验证

```
[训练: 252日] → [测试: 63日] → 滑动63日 → 重复
```

## 七、监控体系

### 7.1 系统健康度

| 维度 | 指标 | 正常范围 |
|-----|------|---------|
| 预测能力 | world_model_mse | 下降趋势 |
| 决策能力 | sharpe_ratio | > 0 |
| 自我优化 | reflection_success_rate | > 60% |
| 热力学一致性 | thermo_violation_count | 下降趋势 |

### 7.2 告警规则

- 世界模型MSE持续上升 → 触发元学习适应
- 夏普比率连续20日为负 → 暂停交易
- 热力学约束违规 > 10次/20日 → 收紧约束

## 八、扩展性设计

### 8.1 数据源扩展

新增数据源只需：
1. 继承 `DataCollector` 基类
2. 实现 `fetch` 方法
3. 在 `UnifiedDataPipeline` 中注册

### 8.2 特征扩展

新增热力学特征：
1. 修改 `ThermoState` 数据类
2. 更新 `ThermoStateEncoder` 输出维度
3. 在配置中调整权重

### 8.3 策略扩展

新增策略类型：
1. 继承策略基类
2. 实现 `select_action` 方法
3. 在回测引擎中注册
